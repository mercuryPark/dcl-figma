Why this exists

$

Free alternative to Figma MCP

Figma's own MCP server requires a paid plan. This plugin gives every LLM coding tool the same context — at no cost.

⌘

Zero network, zero telemetry

networkAccess.allowedDomains: ["none"]. CI enforces it. No analytics, no beacons, no phone-home.

◆

Smart auto-shrink

If Slim exceeds 500 KB, the plugin reduces textSummary, then sectionTree, then drops tokens — each step logged in meta.degraded.

∞

Tool-neutral

Works with Claude Code, Cursor, GitHub Copilot, Aider, ChatGPT — the JSON is just JSON. No vendor lock-in.

Install

No build. No npm. No Figma paid tier.

  1. 01

    Download the release zip

    Grab the latest dcl-figma-vX.Y.Z.zip from GitHub Releases.

    Open Releases →
  2. 02

    Unzip anywhere

    For example ~/figma-plugins/dcl-figma/. The folder contains manifest.json and a dist/ directory.

  3. 03

    Import in Figma Desktop

    Menu → Plugins → Development → Import plugin from manifest… → select the manifest.json from the folder you unzipped.

  4. 04

    Run it

    Plugins → Development → Design Context for LLMs. Done.

Integration snippets

Paste these into your LLM session after attaching the JSON.

I've attached a Figma dump (figma.*.slim.json).

When generating UI code, reference:
- screens[] for layout + copy,
- tokens.colors / tokens.variables for design tokens,
- components[] for reusable patterns.

Do not invent component names that are not in the JSON.
@figma.my-app.home.slim.json

Use this Figma dump as the source of truth for screen
structure, copy, and tokens. Match tokens exactly (hex
and name). Ignore nodes whose `visible` is explicitly
false.
/explain Use the file figma.my-app.home.slim.json as the
design reference. When I ask for a component, map it to
the matching entry in components[] and respect
tokens.typography line heights.
aider figma.my-app.home.slim.json src/
# then in the chat:
# Use the figma.*.slim.json as the canonical UI spec.
# Fields: screens, tokens, components.
Load the JSON file design.slim.json (a Figma design dump).
Its top level contains:
- meta (context about how the file was generated),
- tokens (design tokens: colors, typography, effects, variables),
- screens (per-screen summaries with sectionTree and textSummary),
- components (reusable definitions).

When generating code, mirror token names, respect the
sectionTree hierarchy, and use textSummary strings verbatim.

Zero network. Zero telemetry.

One hard guarantee: your design data never leaves your machine.

  • networkAccess.allowedDomains: ["none"] declared in manifest. Figma blocks all outbound requests at runtime.
  • No analytics SDK, no error reporters, no anonymous metrics.
  • Design data flow: Figma sandbox → plugin UI iframe → <a download> / clipboard. That's the entire surface.
  • CI validates the zero-network invariant on every PR — it cannot silently regress.
  • MIT licensed, auditable line-by-line on GitHub.
